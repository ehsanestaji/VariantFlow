## VCF-Fast Stress-Speed Benchmark

- Generated locally: `2026-05-04`
- Mode: `stress`
- Dataset source: deterministic synthetic stress VCFs generated by `benchmark/generate_stress_vcf.sh`
- Dataset shape: `40` unused INFO fields, `16` samples, `FORMAT=GT:DP:GQ:AD`
- hyperfine: `hyperfine 1.19.0`
- bcftools: `bcftools 1.21`
- Runs: `3`, warmup: `1`

### Command Templates

- VCF-Fast filter: `./target/release/vcf-fast filter <input> --where '<expr>' -o <output>`
- bcftools filter: `bcftools filter -i '<expr>' <input> -o <output>`
- VCF-Fast convert TSV: `./target/release/vcf-fast convert <input> --to tsv -o <output.tsv>`
- bcftools query TSV: `bcftools query -u -f '%CHROM\t%POS\t%ID\t%REF\t%ALT\t%QUAL\t%FILTER\t%INFO/DP\t%INFO/AF\n' <input>`
- VCF-Fast stats: `./target/release/vcf-fast stats <input> > <output.json>`
- bcftools stats: `bcftools stats <input> > <output.txt>`

### Caveats

- Stress data is synthetic and designed to exercise selective parsing under heavy unused INFO/FORMAT/sample payloads.
- The 1M gzip `bcftools` timing had high variance, so the `13.72x` result should be treated as noisy until repeated on a quieter machine.
- `stats` correctness currently compares the overlapping record count field only; richer stats equivalence is a later milestone.
- This report does not prove BCF, BGZF-output, tabix-indexed region reads, or FORMAT-aware predicates.
