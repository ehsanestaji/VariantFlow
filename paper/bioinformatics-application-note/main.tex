\begin{abstract}
\noindent\textbf{Summary:}
Post-calling variant workflows repeatedly scan VCF, BGZF-compressed VCF, and
BCF files for filtering, summary statistics, set comparison, and tabular export.
These operations often need only a small subset of each record, yet mature
general-purpose tools must preserve broad compatibility across many expression
and file-format cases. VariantFlow is a Rust-based command-line engine that
targets selective execution: it parses only fields required by the current
operation, preserves original VCF records where possible, and records
correctness-matched benchmark evidence against bcftools. The current engine
combines byte-slice record views, threaded BGZF input, ordered parallel
predicate evaluation, and TSV/Parquet export. In tracked public and stress
benchmarks, VariantFlow matched normalized bcftools outputs while accelerating
bounded IGSR BGZF filtering, FORMAT-heavy stress predicates, and export-once
DuckDB query workflows. VariantFlow is therefore positioned as a
workflow-specific accelerator and analytical bridge, not a universal replacement
for bcftools, HTSlib, GATK, or VCFtools.

\noindent\textbf{Availability and implementation:}
VariantFlow is implemented in Rust and is released under the MIT OR Apache-2.0
license at \url{https://github.com/ehsanestaji/VariantFlow}. Source release
archives, Bioconda packaging, and Zenodo DOI metadata are prepared as part of
the public release workflow.
