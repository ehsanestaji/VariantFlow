\begin{abstract}
\noindent\textbf{Summary:}
As population-scale projects accumulate variant call sets with thousands of
samples and millions of sites, post-calling analysis has become a recurring
computational bottleneck: workflows repeatedly scan large VCF files, yet most
operations depend on only a few fields per record. Parsing unused annotations,
FORMAT vectors, and thousands of sample values wastes time without changing
the result. We present VariantFlow, whose core design principle is selective
execution: it parses only the fields required by the current predicate,
statistic, or projection, and preserves original records where possible. On
correctness-matched benchmarks using the 1000 Genomes 3\,202-sample dataset,
VariantFlow accelerated missingness computation 3.67--4.78x over VCFtools from
single-chromosome to whole-genome scale at constant 9~MB memory, FORMAT
filtering 4.74--17.78x over bcftools, indexed high-skip predicates 123--273x,
LD 9.88x, and export-once DuckDB queries 1.19--19.20x amortized. By collapsing
each scan to the fields that matter, VariantFlow brings large-cohort
post-calling analysis toward interactive timescales on commodity CPUs,
complementing rather than replacing bcftools, HTSlib, GATK, and VCFtools.

\noindent\textbf{Availability and implementation:}
VariantFlow is open source (Rust) under the MIT OR Apache-2.0
license at \url{https://github.com/ehsanestaji/VariantFlow} and archived at
\href{https://doi.org/10.5281/zenodo.21198171}{doi:10.5281/zenodo.21198171}.
Benchmark reports, reproducibility commands, and supplementary data are in
the repository.

\section*{Figure and table legends}

\noindent\textbf{Table~1. Correctness-matched benchmarks from tracked
reports.} Each row records exact commands, competitor version, correctness
checks, and peak resident memory. IGSR rows use the 1000~Genomes 3202-sample
high-coverage dataset. \checkmark~$=$ byte-identical output vs.\ baseline.
Baselines: \texttt{bcftools filter} (filtering), VCFtools~0.1.17 (population
summaries), repeated scans (DuckDB). See Supplementary Table~S1 for cross-tool
validation including scikit-allel.

\vspace{0.9em}
\noindent\textbf{Figure~1. Selective execution versus conventional
full-record parsing.} (a) Conventional tools decode every field of every VCF
record before applying an operation, so most decoded bytes are discarded.
(b) VariantFlow compiles each query to the minimal set of fields it touches and
decodes only those, preserving the original record. The lower band lists the
population-genetics statistics VariantFlow computes in a single streaming pass;
matrix-wide and haplotype-based statistics are deferred to scikit-allel
(Supplementary Table~S5).

\vspace{0.9em}
\noindent\textbf{Figure~2. The VariantFlow command tree.} Subcommands are
grouped into filtering/indexing/I-O and population-genetics families; each leaf
gives the command and a one-line description of its function.
