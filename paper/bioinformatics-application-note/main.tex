\begin{abstract}
\noindent\textbf{Summary:}
Many post-calling analyses repeatedly scan large VCF, BGZF-compressed VCF, and
BCF files to evaluate filters or export a few columns. These operations often
depend on only QUAL, FILTER, one INFO tag, or one FORMAT field, but conventional
compatibility-first workflows may inspect substantially more of each record.
VariantFlow is a Rust command-line engine for selective execution. It parses
only fields required by the current operation, preserves original VCF records
where possible, and reports speed only for correctness-matched benchmark rows
against bcftools. Its native engine combines borrowed byte-slice record views,
threaded BGZF input, ordered parallel predicate evaluation, and TSV/Parquet
export. On tracked public and stress benchmarks, VariantFlow matched normalized
bcftools outputs while accelerating bounded human CHM13 FORMAT filtering
(4.74--17.78x), IGSR BGZF filtering (13.44--13.47x), CPU-heavy FORMAT
predicates (4.33--5.27x), and export-once DuckDB query workflows
(3.18--25.67x amortized). VariantFlow is therefore a workflow-specific
accelerator and analytical bridge, not a universal replacement for bcftools,
HTSlib, GATK, or VCFtools.

\noindent\textbf{Availability and implementation:}
VariantFlow is implemented in Rust and is released under the MIT OR Apache-2.0
license at \url{https://github.com/ehsanestaji/VariantFlow}. Source release
archives, Bioconda packaging, and Zenodo DOI metadata are prepared as part of
the public release workflow.
