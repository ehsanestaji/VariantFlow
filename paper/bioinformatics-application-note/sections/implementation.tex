\section{Algorithm and implementation}

The core design principle is selective execution: a query is compiled to the
minimal set of fields it touches, and only those fields are decoded
(Figure~1), in contrast to the conventional approach of decoding
every field of every record. The
native engine treats each VCF record as a borrowed byte-slice view. Site
fields, selected INFO values, FORMAT keys, and sample values are located by
byte scanning rather than by materializing all columns as strings. Expression
parsing produces a typed predicate tree, and the evaluator asks the record
view only for fields required by that predicate. This keeps the hot path close
to the work requested by the user.

For FORMAT-heavy cohort filters, VariantFlow scans the FORMAT schema once per
record, caches the requested key index, and reads sample values by index. This
avoids repeatedly scanning \texttt{GT:DP:GQ:AD}-like schemas for every sample.
Native filtering writes passing records in input order and preserves their
original text in the line-preserving path.

Compressed input is handled through a native BGZF reader with auto-capped
worker threads. CPU-heavy predicates use ordered batch evaluation while
preserving output order. For repeated high-skip filters, an explicit
\texttt{.vfi} sidecar index stores chunk metadata (genomic range, QUAL,
FILTER, selected INFO summaries); the planner skips chunks only when metadata
proves no record can pass. Optional HTSlib-backed paths provide BCF input,
indexed region reads, and BGZF output.

VariantFlow also provides population-genetics and export workflows. The
VCFtools-style commands use compact diploid biallelic genotype summaries where
supported, preserving VCFtools-compatible output for frequency, missingness,
HWE, LD, Fst, and related population summaries. Parquet~\citep{parquet} export
stores typed columns for repeated analysis with DuckDB \citep{duckdb}.

The CLI follows the small-tool style common in genomics, organised as a small
set of composable subcommands (Figure~2):
{\small
\texttt{variantflow filter input.vcf.gz --where "QUAL>30 \&\& ANY(FORMAT/DP>20)"}\\
\texttt{variantflow fst cohort.vcf.gz --pop pop1.txt --pop pop2.txt -o fst.tsv}
}
