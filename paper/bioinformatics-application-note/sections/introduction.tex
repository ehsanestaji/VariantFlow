\section{Introduction}

Genomic variant data is accumulating rapidly. Population-scale projects now
routinely produce VCF files with thousands of samples and millions of variant
sites per chromosome \citep{vcftools}. Post-calling workflows---quality
filtering, cohort subsetting, population summaries, analytical export---scan
these files repeatedly, and each pass typically parses the full record even
when the operation depends on only QUAL, one INFO tag, or a single FORMAT
key. On a multi-thousand-sample cohort, parsing unused FORMAT vectors and
sample values can dominate runtime without changing the result.

Mature tools---bcftools, HTSlib, VCFtools---form the reliable
infrastructure on which larger pipelines rely
\citep{bcftools,htslib}. Their breadth is essential, but an exchange
format optimized for generality is not always the most efficient
execution layout for every downstream task.

VariantFlow targets this through selective execution: parse only the
fields required by the current predicate, statistic, or projection;
preserve original records where possible; and track every performance
claim against correctness-matched benchmarks. It complements bcftools,
HTSlib, VCFtools, and GATK rather than replacing them.
